## 生成任务清单

使用 `/plan` SKILL 基于指定的需求文档生成开发的工作任务, 在任务完成后显示任务概览

###  任务要求

- 生成可检查的任务清单
- 开发完毕通过 `/code-review` skill 进行代码审查

### 单测要求
- 满足项目单测覆盖率要求
- 单元测试的标签使用中文

### 保存位置

保存到当前需求目录下的 `tasks.md`

### 更新文档

任务整体开发并验收完毕后更新相关的设计文档 docs/{service-name}/[business|flows|overview|suggest].md 文件

- 读取 `docs/{service-name}/overview.md`, 确定影响模块
- 确定需要更新的文档文件, 辅助更新内容

### 依赖关系

- 描述执行任务的依赖关系
- 描述任务的执行关系, 给 Agent 并行做参考

## 任务模板

> 按照以下任务模版进行输出

```md
# {需求文档标题} 任务清单

## 任务状态

- [ ] 待开始
- [~] 进行中
- [x] 已完成

## Agent 执行约定

> 以下约定对执行本任务清单的 Agent 有约束力。

- **开始子任务**：将对应行的 `- [ ]` 改为 `- [~]`（进行中）
- **完成子任务**：将对应行的 `- [~]` 改为 `- [x]`（已完成）
- **粒度(必须)**：每完成一个叶子任务（如 `1.1`、`2.3`）立即更新该行，不要等到阶段结束
- **不可修改**：不要修改 Agent 约定块本身、任务编号和任务描述文字，只修改 `[ ]` / `[~]` / `[x]`
状态标记

## 1. {开发任务分组 1}

- [ ] **1.1** 开发任务 1
- [ ] **1.2** ...

## 2. {开发任务分组 2}

...

## 3. 编写单元测试

> 单测覆盖率要求 ≥ {百分比}%（遵循架构规则）
> 其他单测要求 : ...

- [ ] **3.1** {测试用例 1}
- [ ] **3.2** ...


## 4. 验证与审查

- [ ] **4.1** 运行 `/code-review` skill 审查全部 diff，修复发现的问题
- [ ] **4.2** ...

## 5. 文档更新

- [ ] **7.1** 更新 `docs/xx/xx.md` — 更新的概述


## 任务依赖关系

- 执行顺序 : 1 (工作内容) → 2, 3, 4 (工作项目) → 5 (测试) → 6 (验证) → 7 (文档)
- 依赖关系 : 任务 2、3、4 可并行开发（均依赖任务 1 完成）
- 其他约束 : ....
```
