# 云效需求批量导入

## 快速执行流程（按顺序执行）

### Step 0: 读取配置（必做）
从 CLAUDE.md 读取：
- `organization_id`: 组织 ID

### Step 1: 读取并解析 Excel
使用 Python 读取 Excel 文件，自动识别表头，提取以下字段：
- 需求标题
- 负责人姓名
- 需求类型
- 预期收益/价值
- 需求来源/反馈人
- 归属模块
- 完整需求描述

### Step 2: 项目选择（必须执行，且在创建前完成）
**⚠️ 重要：必须让用户选择项目，确认后才能开始创建需求！**

调用项目列表接口并显示给用户选择：
```python
mcp__yunxiao__search_projects(organizationId="<组织 ID>")
```

以表格形式显示项目列表，让用户选择序号。用户确认选择后才能继续下一步。

### Step 3: 固定映射表（直接使用，无需查询）

**负责人 -> 用户 ID 映射：**
```
任广焱 -> 6836b16013953752274d9500
王海洋 -> 665d6b2be250d584b90a9dbc
其他/空 -> 65699cacc319d2a0f949d3a7（当前用户）
```

**自定义字段 ID 映射：**
```
反馈人 -> e3e6f16092d2b98b1a85026e39
```

**需求类型 ID 映射：**
```
产品类需求 -> 9uy29901re573f561d69jn40
技术类需求 -> bca48ee2a0976d38f4802fae
```

### Step 4: 批量创建工作项

对每条需求直接调用创建接口：

```python
mcp__yunxiao__create_work_item(
  organizationId="<organization_id>",
  spaceId="<用户选择的项目 ID>",
  subject="<需求标题>",
  workitemTypeId="<需求类型 ID>",
  assignedTo="<负责人 ID>",
  customFieldValues={"e3e6f16092d2b98b1a85026e39": "<反馈人姓名>"},
  description="<HTML 格式描述>"
)
```

### Step 5: 描述内容固定格式（直接生成 HTML）

```html
<h3>为什么要做</h3>
<h4>业务背景</h4>
<p>需求完整描述内容</p>
<h4>业务价值 / 预期收益</h4>
<p>预期收益内容（无则不显示本段）</p>
<hr />
<h3>业务规则</h3>
<ul>
<li><strong>业务场景 / 模块</strong>：归属模块</li>
<li><strong>需求类型</strong>：体验/增长/效率/稳定/策略</li>
<li><strong>需求来源</strong>：反馈人姓名</li>
</ul>
<hr />
<h3>需求描述</h3>
<p>需求完整描述</p>
<hr />
<h3>验收标准</h3>
<ul>
<li>核心功能正常上线</li>
<li>达到预期业务目标</li>
<li>相关页面和流程正常运行</li>
</ul>
```

### Step 6: 导入完成报告

完成后输出：

```

━━━━━━━━━━━━━━━━━━━━━
总计：N 条
成功：N 条
失败：0 条
```

## 注意事项

1. **必须先选择项目**：用户触发导入后，必须先执行 Step 2 让用户选择项目并确认，**绝对不能直接开始创建**，只有用户确认项目后才能开始批量创建
2. **错误处理**：单条创建失败时记录并继续，不中断整个批量导入
3. **限速**：连续创建间隔 0.5 秒，避免触发限流
4. **空值处理**：无反馈人时 customFieldValues 字段值为空字符串；无负责人时使用默认用户 ID
5. **描述格式**：固定使用 HTML 标签格式，不使用 Markdown
