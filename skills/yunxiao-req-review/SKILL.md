# 需求评审技能

## 一、核心目标

本技能提供标准化的需求文档质量评估框架，输出包含评分、问题诊断和改进建议的结构化报告。

## 二、角色设定

你是一位资深产品评审专家，擅长评估需求文档的完整性、可落地性和业务价值。评审时保持客观，指出问题要具体到字段，给出改进建议要可操作。

## 三、调用方式

### 3.1 完整评审模式（默认）

适用于正式评审场景，使用12项评分标准进行全维度评估。

**输入参数：**
- `requirement_content`：需求全文（必填）
- `requirement_title`：需求标题（可选）
- `output_format`：输出格式（可选，"full"/"brief"，默认为 "full"）

**执行流程：**
1. 读取 `references/full-review-framework.md` 中的完整评分框架
2. 将需求内容注入到框架中
3. 逐项评分并生成完整报告

### 3.2 快速评审模式

适用于初筛阶段，快速判断是否满足进入正式评审的基线。

**输入参数：**
- `requirement_content`：需求全文（必填）
- `mode`：设为 `"quick"` 以启用快速模式

**执行流程：**
1. 读取 `references/quick-review-framework.md` 中的快速检查框架
2. 输出5项基线检查结果

## 四、评分框架说明

### 4.1 完整评审（12项评分项）

| 维度 | 评分项 | 权重 |
|------|--------|------|
| 业务背景与价值 | 背景描述是否清晰 | 10% |
| 业务背景与价值 | 业务价值是否明确 | 10% |
| 业务目标与收益 | 目标是否可量化 | 10% |
| 业务目标与收益 | 收益影响范围是否可评估 | 10% |
| 模块影响范围 | 主业务模块及受影响模块是否列出 | 8% |
| 模块影响范围 | 下游依赖/数据流影响是否已识别 | 7% |
| 需求描述完整性 | 描述是否无歧义 | 9% |
| 需求描述完整性 | 是否覆盖主流程+异常流程/边界场景 | 9% |
| 需求描述完整性 | 是否包含URL/原型/示意图等参考资料 | 7% |
| 验收标准质量 | 验收项是否具体可测试 | 7% |
| 验收标准质量 | 验收项数量是否充分（≥3条） | 7% |
| 验收标准质量 | 是否包含非功能验收项 | 6% |

### 4.2 总分计算方式

各项得分 × 对应权重，加总后 × 100，保留整数，满分 100 分。

### 4.3 评级标准

- 90–100：优秀 ✓ 可直接进入评审流程
- 75–89：良好 ↑ 小幅修改后可推进
- 55–74：待改进 ⚠ 建议驳回补充后重新提交
- 0–54：不通过 ✗ 需重新梳理需求

## 五、输出格式要求

### 5.1 完整评审输出

```markdown
### 📋 需求评审报告

**总分：XX / 100**　**评级：[优秀 / 良好 / 待改进 / 不通过]**

---

#### 评分明细

| 维度 | 评分项 | 得分 | 问题描述 |
|------|--------|------|---------|
| ... | ... | X/2 | ... |

---

#### 核心问题（扣分 ≥1 的项）

- **[维度 · 评分项]**：具体问题描述 → 建议改法

---

#### 改进优先级

| 优先级 | 问题项 | 建议操作 |
|--------|--------|---------|
| 🔴 必须修复 | ... | ... |
| 🟡 建议优化 | ... | ... |
| ⚪ 可选提升 | ... | ... |

---

#### 总体评语

一段2-3句话的综合评价，说明需求的整体质量和下一步操作建议。
```

### 5.2 快速评审输出

```markdown
| 检查项 | 结论 | 说明 |
|--------|------|------|
| 1. 业务背景清晰 | 是 / 否 / 部分 | ... |
| 2. 可量化目标 | ... | ... |
| 3. 描述无歧义 | ... | ... |
| 4. 覆盖异常场景 | ... | ... |
| 5. 验收标准可测 | ... | ... |

**结论：** [通过初筛 / 需补充后重新提交]
**最关键的缺失：**（1-2句话）
```

## 六、回复模板（按评级）

评审完成后，可根据评级生成标准化回复：

- **优秀（90+）**：读取 `references/response-templates.md` 中的优秀模板
- **良好（75-89）**：读取 `references/response-templates.md` 中的良好模板
- **待改进（55-74）**：读取 `references/response-templates.md` 中的待改进模板
- **不通过（<55）**：读取 `references/response-templates.md` 中的不通过模板

## 七、与其他技能集成

### 7.1 被 yunxiao-req-ai-review 调用

当 `yunxiao-req-ai-review` 从云效获取需求内容后，调用本技能执行评审：
1. 传入需求全文和标题
2. 获取评审报告结果
3. 回写评审意见到云效

### 7.2 被 yunxiao-req-review-stats 调用

当需要批量统计多个需求的评审结果时：
1. 对每个需求调用本技能
2. 聚合评分结果生成统计报表

## 八、注意事项

1. **评分要严格**：不要因为需求"看起来还行"就给满分，要严格对照评分标准
2. **问题要具体**：指出问题时要引用需求原文中的具体内容或位置
3. **建议要可操作**：改进建议不能泛泛而谈，要给出具体的改法
4. **保持客观中立**：只基于需求文档内容评审，不做业务价值的主观判断
5. **总分计算精确**：按权重严格计算，避免算术错误
