# 云效未排期需求导出

## 技能概述

从云效指定项目中导出「没有迭代（sprint为空）」且「状态未完成」的需求，支持选择导出产品类需求或技术类需求，支持多选标签筛选（包含任意一个标签即可），并以 JSON 格式输出结果。

## 触发条件

当用户出现以下表述时必须使用此技能：
- "导出未排期需求"
- "导出没有迭代的需求"
- "导出未完成的产品需求"
- "导出不在迭代中的需求"
- "导出产品需求列表"
- "查看未排期的产品需求"

## 使用流程

### 步骤 1：交互式确认参数

使用 `AskUserQuestion` 工具交互式询问用户以下参数：

| 参数 | 说明 | 必填 | 默认值 | 询问方式 |
|------|------|------|--------|----------|
| `project_name` | 项目名称 | 是 | - | 单选，列出用户可见的项目供选择 |
| `req_type` | 需求类型 | 是 | product | 单选：产品类需求 / 技术类需求 |
| `filter_tags` | 筛选标签 | 否 | [] | 多选：从项目现有标签中选择，包含任意一个标签即可 |
| `filter_by_sprint` | 是否按迭代筛选 | 是 | true | 单选：仅导出无迭代需求 / 包含有迭代需求 |
| `output_file` | 输出文件路径 | 否 | `unplanned_requirements.json` | 文本输入 |

**注意**：
1. 先查询项目列表，让用户选择项目
2. 选择项目后，查询该项目的所有标签供用户多选
3. 标签筛选逻辑：包含任意一个选中的标签即匹配
4. 如果用户已在对话中明确了参数，可以跳过对应询问

### 步骤 2：查询云效工作项

使用 `mcp__yunxiao__search_workitems` 工具查询所有需求：

```typescript
{
  organizationId: "从 CLAUDE.md 读取或用户提供",
  spaceId: "用户提供的项目 ID",
  category: "Req",           // 需求类型
  includeDetails: true,
  perPage: 200,
  orderBy: "gmtCreate",
  sort: "desc"
}
```

> **注意**：所有产品类和技术类需求的 category 都是 Req，在本地筛选时通过 `workitemType.name` 来区分具体类型。

### 步骤 3：自动分页处理

当返回结果的总数超过 200 条时，需要自动处理分页：

1. 从第一页结果中获取 `total` 总数
2. 计算总页数：`Math.ceil(total / 200)`
3. 循环请求第 2 页到最后一页，将所有结果合并

### 步骤 4：本地筛选条件

获取所有结果后，在本地进行二次筛选：

**筛选条件 1：没有迭代**
```javascript
sprint === null || sprint === undefined || sprint === "" || !sprint.id
```

**筛选条件 2：需求类型过滤**  
根据用户选择的类型，通过 `workitemType.name` 进行本地过滤：
- **产品类需求**：`workitemType.name == "产品类需求"` 或包含"产品"关键词
- **技术类需求**：`workitemType.name == "技术类需求"` 或包含"技术"关键词

**筛选条件 3：状态未完成**  
排除以下已完成状态（根据云效系统常见配置）：
- 状态名称包含"已完成"、"已关闭"、"已验收"、"已发布"、"已上线"
- 状态 ID 不在常见的未完成状态列表中

常见未完成状态 ID：
- 28: 待确认
- 100005: 待处理
- 30: 重新打开
- 34: 延期处理
- 32: 已确认
- 625489: 已选中
- 154395: 分析中
- 165115: 分析完成
- 100010: 进行中
- 156603: 设计中
- 307012: 设计完成
- 142838: 开发中
- 100011: 开发完成
- 100012: 测试中

### 步骤 5：输出结果

将筛选后的结果保存为 JSON 文件，格式如下：

```json
{
  "export_time": "2024-01-15T10:30:00.000Z",
  "project_id": "项目ID",
  "project_name": "项目名称",
  "req_type": "product/tech",
  "req_type_name": "产品类需求/技术类需求",
  "total_items": 42,
  "items": [
    {
      "id": "工作项ID",
      "subject": "标题",
      "status": "状态名称",
      "status_id": "状态ID",
      "priority": "优先级",
      "assigned_to": "指派人",
      "creator": "创建人",
      "created_at": "创建时间",
      "description": "详细描述",
      "sprint": null,

      "category": "Req",
      "version": "所属版本",

    }

}
```

### 步骤 6：向用户汇报

输出完成后，向用户汇报：
- 导出的需求总数
- 输出文件的路径
- 按状态分组的统计（可选）

## 注意事项

1. **保留全部字段**：不要过滤原始返回的字段，用户需要完整数据
2. **分页必须处理**：不要只导出第一页的结果
3. **需求类型确认**：执行前必须确认用户需要产品类需求还是技术类需求
4. **错误处理**：如果 API 返回错误，将错误信息清晰地告诉用户
5. **空结果处理**：如果没有符合条件的需求，告知用户"该项目中没有未排期且未完成的{产品/技术}类需求"
