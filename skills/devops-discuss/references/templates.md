# 文档模板与输出格式

保存讨论记录时，根据讨论模式选择对应模板。

---

## 完整讨论模式模板

```markdown
# {变更主题}需求变更讨论

## 需求背景

{业务背景描述，包括谁提出、解决什么问题}

## 讨论后的关键结论

{用要点列表列出本次讨论最重要的结论，放在最前面方便快速阅读}

## 需求目标

{2-3 句话描述变更要达成的业务目标和边界}

## 当前流程

{描述当前系统处理流程}

参考：
- {列出引用的架构文档路径}

## 影响分析

### 1. {服务名}

- {具体影响点}
- {具体影响点}

### 2. {服务名}

- {具体影响点}

### 3. 级联副作用

{事件链、异步处理等影响}

### 4. 数据一致性与过渡

{一致性风险和过渡方案}

## 方案对比

### 方案 A：{方案名}

{核心思路}

优点：
- {优点}

缺点：
- {缺点}

### 方案 B：{方案名}（如有）

{同上格式}

## 推荐方案

{最终倾向的方案及理由}

## 待确认事项

- {待确认点}

## 实施建议

{落地建议，按优先级排列}

## 结论

{一段话总结这次变更的本质和关键决策}
```

## 输出确认格式

**完整讨论模式：**
```
✅ 讨论记录已保存

文件：docs/discuss/{业务域}/{变更主题}/discuss.md
模式：完整讨论
业务域：{业务域}
变更主题：{变更主题}
方案倾向：{最终倾向的方案}
待确认项：{N} 项
```

**影响分析模式：**
```
✅ 影响分析已保存

文件：docs/discuss/{业务域}/{变更主题}/discuss.md
模式：影响分析
业务域：{业务域}
变更主题：{变更主题}
影响服务：{受影响的服务列表}
待确认项：{N} 项
```
